# LLM Wiki Index

This folder is the agent-readable wiki for the project. It turns scattered project knowledge into durable pages that Codex, Claude Code, Cursor, Gemini CLI, and future agents can read before editing.

## Map

| Page | Purpose |
| --- | --- |
| `llm-wiki.md` | Explains the LLM Wiki pattern and how this repo uses it. |
| `project-overview.md` | Defines the repo mission, audience, and outcomes. |
| `architecture.md` | Explains the static site structure and data flow. |
| `design-principles.md` | Captures visual and UX rules for the public site. |
| `skill-review-rubric.md` | Defines review scoring, status labels, and data expectations. |
| `testing-and-release.md` | Lists validation commands and release checks. |
| `github-workflow.md` | Captures branch, issue, PR, and commit expectations. |
| `hermes-ai-notes.md` | Explains where Hermes-style agents can help and where to be careful. |

## How Agents Should Use This

1. Search the wiki for the task topic.
2. Read the smallest relevant page set.
3. Make the repo change.
4. Add any reusable lesson back to the wiki.

The wiki is not a diary. It is a working memory layer for repeatable project knowledge.
