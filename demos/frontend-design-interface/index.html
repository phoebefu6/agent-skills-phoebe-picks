Key features

The Skill turns a brief into a real screen.

Each control maps to one frontend-design concept used in the build.

01

Understand the job before the look.

  • Audience: AI-curious builders who need a practical Skill finder.
  • Job: browse, compare, and decide which Skill to try.
  • Constraint: static HTML, CSS, and JavaScript only.
02

Commit to one concrete direction.

  • Direction: calm editorial product console.
  • Palette: low-saturation paper, green accents, warm proof chips.
  • Layout: product screen first, no generic marketing poster.
03

Design states a real user expects.

  • Ready state with actual Skill rows.
  • Loading state for data fetch or refresh.
  • Empty and error states with clear recovery copy.
04

Use semantic, responsive frontend code.

  • Plain static files fit the repo architecture.
  • Controls expose focus states and button state changes.
  • Grid collapses cleanly on tablet and mobile.
05

Make the product memorable without noise.

  • Distinct console frame and source chips.
  • Alignment, density, and spacing support scanning.
  • Motion stays tied to state changes.
06

Check before calling it done.

  • Desktop and mobile layouts are both meaningful.
  • Text does not rely on placeholder fluff.
  • The screen has one clear memory: Skills by situation.
