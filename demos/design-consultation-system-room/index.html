Key features

Consultation turns taste into reusable rules.

The wrapper is a kickoff Skill, so the demo becomes a workshop board rather than a final landing page.

01

Start from product truth.

  • Audience: followers who want useful Skills fast.
  • Promise: proof-first curation.
  • Boundary: internal SOP stays out of the public page.
02

Create reusable tokens.

  • Color roles, not random swatches.
  • Typography hierarchy for gallery and demo pages.
  • Component patterns for cards, chips, drawers, and proof links.
03

Name creative risks early.

  • Too dry: followers lose interest.
  • Too decorative: proof gets buried.
  • Too much method: the page feels like homework.
04

Show realistic product surfaces.

  • Desktop gallery mockup.
  • Mobile Skill card mockup.
  • Proof drawer mockup.
05

Make it useful for future agents.

  • Rules are short enough to reuse.
  • Design choices connect to audience value.
  • Tokens and mockups guide future pages.
