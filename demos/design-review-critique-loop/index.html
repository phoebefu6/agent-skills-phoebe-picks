Key features

Review is a loop, not an opinion.

The wrapper is compact, so this demo makes the implied workflow explicit: audit, prioritize, fix, compare, ship.

01

Start with what the eye sees.

  • Scan hierarchy, spacing, contrast, alignment, copy, and mobile risk.
  • Name the actual visual issue instead of saying it looks off.
  • Connect each issue to follower comprehension or trust.
02

Rank issues before editing.

  • Critical: blocks understanding or action.
  • Major: weakens trust or makes scanning harder.
  • Minor: polish work after the page is clear.
03

Make each fix atomic.

  • One visual problem maps to one focused change.
  • Fixes are small enough to test and revert.
  • Design critique becomes a practical work queue.
04

Show the before and after.

  • Followers can see what changed without reading a long method note.
  • Before-after comparison keeps the review honest.
  • Visual evidence is stronger than abstract design talk.
05

Decide if the page is ready.

  • Do not keep polishing forever.
  • Ship when comprehension, trust, and action are clear.
  • Keep the remaining polish list visible.
